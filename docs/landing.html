Why depguard?

Every npm install is a trust decision. depguard gives you the data to make it.

🛡

Vulnerability Triage

Combines npm Registry and GitHub Advisory Database. Deduplicates, filters by your installed version, and suggests specific upgrade paths to fix CVEs.

⛓

Supply Chain Attack Detection

Static analysis of install scripts catches reverse shells, credential exfiltration, obfuscated code, and remote code execution patterns before they run.

📊

Package Scoring (0-100)

Weighted scoring across security, maintenance, popularity, license compatibility, and dependency count. Configurable weights for your risk profile.

🤖

AI-Native (MCP Server)

Built-in MCP server lets any AI agent — Claude, Cursor, Windsurf, Continue.dev — audit packages during code generation. Token savings tracked per response.

💡

Smart Advisor

"Should I install this?" — depguard checks native Node.js alternatives first, then scores npm candidates, and recommends install, caution, or write-from-scratch.

📜

License Compliance

15+ SPDX licenses ranked by permissiveness. Instantly know if a dependency's license is compatible with your project — no lawyer required.

Use Cases

From solo devs to enterprise security teams.

AI Agent Guardrail

Your AI coding assistant wants to npm install a package. depguard audits it first and blocks risky installs automatically.

// AI agent calls MCP tool before installing const audit = await depguard_audit("some-package") if (audit.score < 60) reject("Package too risky")

CI/CD Pipeline Gate

Block PRs that add vulnerable or unmaintained dependencies. Audit the entire package.json on every push.

depguard-cli audit-project ./package.json --json \ | jq '.vulnerable > 0' && exit 1

Dependency Review

Before adding any new dependency, get a data-driven answer: install, write from scratch, or use the native Node.js alternative.

depguard-cli should-use "yaml parser" --threshold 70 # Action: install # Package: yaml (score 88/100)

Security Audit Report

Generate a full vulnerability and license compliance report for all project dependencies in one command.

depguard-cli audit-project ./package.json # Total: 47 packages # Vulnerable: 3 (2 high, 1 moderate) # Fix suggestions included

Design Principles

Security tooling should practice what it preaches.

Zero Dependencies

Only Node.js built-in APIs. No supply chain risk from the security tool itself.

Never Throws

Network failures return degraded results with warnings — never crashes your pipeline.

TypeScript Strict

Full type safety. Every API response is typed. Every edge case handled at compile time.

100% Offline Tests

All 93 tests use mock fetch. CI never flakes due to network issues.
